# 피하기 게임 requirement
## 목적
- 플레이어가 마우스 우클릭으로 이동하면서 공격을 피하는 생존형 점수 게임을 만든다.
- 캐릭터는 고정되고 배경이 움직이는 방식으로 이동감을 연출하여 실감나는 플레이를 제공한다.
- 시간이 지날수록 난이도가 상승하고 다양한 공격 패턴이 등장해 긴장감을 유지한다.
- 플레이어는 이름과 점수를 남겨 랭킹을 관리할 수 있다.

## 필수 기능
- 마우스 우클릭으로 캐릭터 이동 목표를 지정한다.
- 캐릭터는 실제로는 고정된 위치에 있고 배경과 적 공격이 움직이는 형태로 화면 내 이동감을 구현한다.
- 캐릭터는 앞, 뒤, 좌, 우, 대각선 8방향 애니메이션 또는 방향 표현을 지원한다.
- 여러 종류의 공격이 등장하여 플레이어를 위협한다.
- 공격에 캐릭터가 맞으면 즉시 사망하고 게임이 처음부터 다시 시작된다.
- 시간이 흐를수록 난이도가 올라가며 공격 수와 속도가 증가한다.
- 난이도별 배경 테마를 적용한다: 쉬움은 초원, 보통은 정글, 어려움은 지옥.
- 플레이어는 게임 종료 후 이름을 입력하고 점수를 저장해 랭킹을 확인할 수 있다.

## 세부 기능
- 우클릭 시 목표 지점이 표시되고, 배경/적이 반대 방향으로 움직이며 캐릭터 이동처럼 보인다.
- 8방향 캐릭터 모습은 방향별 이미지, 색상 변화, 회전 또는 스프라이트 애니메이션으로 표현한다.
- 공격 유형 예시:
  - 직선 투사체 공격
  - 플레이어를 추적하는 적
  - 특정 위치에 생성되는 지뢰 또는 장애물
  - 넓은 범위 공격(광역 폭발 등)
- 난이도 상승 방식:
  - 시간이 지남에 따라 공격 생성 빈도 증가
  - 중간 보스급 공격 또는 복합 공격 패턴 추가
  - 적 공격 속도 및 이동 범위 확장
- 점수 산정 방식:
  - 생존 시간 기반 점수 증가
  - 공격 회피 보너스
  - 난이도별 점수 보정
- 플레이어가 죽으면 재시작 화면으로 돌아가며, 이름과 점수를 입력할 수 있는 랭킹 등록창을 표시한다.
- 랭킹은 이름+점수 형식으로 브라우저 `localStorage`에 저장하여 세션 간 유지한다.
- 랭킹 화면에 상위 점수를 최대 10개까지 표시한다.

## 디자인 요구
- 리얼한 느낌의 2D 그래픽 스타일을 사용한다.
- 배경은 난이도별로 명확히 구분되며, 초원/정글/지옥의 질감과 색감이 살아있어야 한다.
- 캐릭터 및 공격 이펙트는 사실적인 라이팅, 쉐이딩, 움직임을 반영한다.
- UI는 게임 상태, 점수, 남은 시간/난이도, 랭킹 입력창을 직관적으로 보여준다.
- 데스/리스타트 시 전환 애니메이션 또는 페이드 효과를 적용해 몰입감을 높인다.

## 기술 스택
- HTML5
- CSS3
- JavaScript (ES6+)
- Canvas API (2D Context)
- Browser `localStorage`
